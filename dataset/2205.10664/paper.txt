::: ltx_abstract
###### Abstract {#abstract .ltx_title .ltx_title_abstract}

Temporal domain generalization is a promising yet extremely challenging
area where the goal is to learn models under temporally changing data
distributions and generalize to unseen data distributions following the
trends of the change. The advancement of this area is challenged by: 1)
characterizing data distribution drift and its impacts on models, 2)
expressiveness in tracking the model dynamics, and 3) theoretical
guarantee on the performance. To address them, we propose a Temporal
Domain Generalization with Drift-Aware Dynamic Neural Network (DRAIN)
framework. Specifically, we formulate the problem into a Bayesian
framework that jointly models the relation between data and model
dynamics. We then build a recurrent graph generation scenario to
characterize the dynamic graph-structured neural networks learned across
different time points. It captures the temporal drift of model
parameters and data distributions and can predict models in the future
without the presence of future data. In addition, we explore theoretical
guarantees of the model performance under the challenging temporal DG
setting and provide theoretical analysis, including uncertainty and
generalization error. Finally, extensive experiments on several
real-world benchmarks with temporal drift demonstrate the proposed
method's effectiveness and efficiency.
:::

## [1 ]{.ltx_tag .ltx_tag_section}Introduction {#introduction .ltx_title .ltx_title_section}

::: {#S1.p1 .ltx_para .ltx_noindent}
In machine learning, researchers often assume that training and test
data follow the same distribution for the trained model to work on test
data with some generalizability. However, in reality, this assumption
usually cannot be satisfied, and when we cannot make sure the trained
model is always applied in the same domain where it was trained. This
motivates Domain Adaptation (DA) which builds the bridge between source
and target domains by characterizing the transformation between the data
from these
domains \[[3](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib3){.ltx_ref},
[10](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib10){.ltx_ref},
[35](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib35){.ltx_ref}\].
However, in more challenging situations when target domain data is
unavailable (e.g., no data from an unknown area, no data from the
future, etc.), we need a more realistic scenario named Domain
Generalization
(DG) \[[32](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib32){.ltx_ref},
[1](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib1){.ltx_ref},
[7](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib7){.ltx_ref}\].
:::

::: {#S1.p2 .ltx_para .ltx_noindent}
Most existing works in DG focus on generalization among domains with
categorical indices, such as generalizing the trained model from one
dataset (e.g.,
MNIST \[[18](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib18){.ltx_ref}\])
to another (e.g.,
SVHN \[[29](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib29){.ltx_ref}\]),
from one task (e.g., image
classification \[[15](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib15){.ltx_ref}\])
to another (e.g., image
segmentation \[[22](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib22){.ltx_ref}\]),
etc. However, in many real-world applications, the "boundary" among
different domains is unavailable and difficult to detect, leading to a
*concept drift* across the domains. For example, when a bank leverages a
model to predict whether a person will be a "defaulted borrower",
features like "annual incoming", "profession type", and "marital status"
are considered. However, due to the temporal change of the society, how
these feature values indicate the prediction output should change
accordingly following some trends that could be predicted somehow in a
range of time. Figure [[1]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#S1.F1 "Figure 1 ‣ 1 Introduction ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}
shows another example, seasonal flu prediction via Twitter data which
evolves each year in many aspects. For example, monthly active users are
increasing, new friendships are formed, the age distribution is shifting
under some trends, etc. Such temporal change in data distribution
gradually outdated the models. Correspondingly, suppose there was an
ideal, always update-to-date model, then the model parameters should
gradually change correspondingly to counter the trend of data
distribution shifting across time. It can also "predict" what the model
parameters should look like in an arbitrary (not too far) future time
point. This requires the power of *temporal* domain generalization.
:::

::: {#S1.p3 .ltx_para .ltx_noindent}
However, as an extension of traditional DG, temporal DG is extremely
challenging yet promising. Existing DG methods that treat the domain
indices as a categorical variable may not be suitable for temporal DG as
they require the domain boundary as apriori to learn the mapping from
source to target
domains \[[27](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib27){.ltx_ref},
[26](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib26){.ltx_ref},
[2](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib2){.ltx_ref},
[1](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib1){.ltx_ref}\].
Until now, temporal domain indices have been well explored only in
DA \[[13](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib13){.ltx_ref},
[30](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib30){.ltx_ref},
[36](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib36){.ltx_ref}\]
but not DG. There are very few existing works in temporal DG due to its
big challenges. One relevant work is *Sequential Learning Domain
Generalization*
(S-MLDG) \[[19](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib19){.ltx_ref}\]
that proposed a DG framework over sequential domains via
meta-learning \[[9](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib9){.ltx_ref}\].
S-MLDG meta-trains the target model on all possible permutations of
source domains, with one source domain left for meta-test. However,
S-MLDG in fact still treats domain index as a categorical variable, and
the method was only tested on categorical DG dataset. A more recent
paper called *Gradient Interpolation*
(GI) \[[28](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib28){.ltx_ref}\]
proposes a temporal DG algorithm to encourage a model to learn functions
that can extrapolate to the near future by supervising the first-order
Taylor expansion of the learned function. However, GI has very limited
power in characterizing model dynamics because it can only learn how the
activation function changes along time while making all the remaining
parameters fixed across time.
:::

::: {#S1.p4 .ltx_para .ltx_noindent}
The advancement of temporal domain generalization is challenged by
several critical bottlenecks, including [1) Difficulty in characterizing
the data distribution drift and its influences on models.]{#S1.p4.1.1
.ltx_text .ltx_font_bold} Modeling the temporally evolving distributions
requires making the model time-sensitive. Intuitive ways include feeding
the time as an input feature to the model, which is well deemed simple
yet problematic as it discards the other features' dependency on time
and dependency on other *confounding* factors changed along
time \[[36](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib36){.ltx_ref}\].
Another possible way is to make the model parameters a function of time.
However, these ways cannot generalize the model to future data as long
as the whole model's dynamics and data dynamics are not holistically
modeled. [2) Lack of expressiveness in tracking the model
dynamics.]{#S1.p4.1.3 .ltx_text .ltx_font_bold} Nowadays, complex tasks
have witnessed the success of big complex models (e.g., large
CNNs \[[6](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib6){.ltx_ref}\]),
where the neurons and model parameters are connected as a complex graph
structure. However, they also significantly challenge tracking their
model dynamics in temporal DG. An expressive model dynamics
characterization and prediction requires mapping data dynamics to *model
dynamics* and hence the graph dynamics of model parameters across time.
This is a highly open problem, especially for the temporal DG area. [3)
Difficulty in theoretical guarantee on the performance.]{#S1.p4.1.5
.ltx_text .ltx_font_bold} While there are fruitful theoretical analyses
on machine learning problems with
independent-and-identically-distributed (*i.i.d*) assumptions (e.g.,
generalization error
bound \[[12](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib12){.ltx_ref}\]),
similar analyses meet substantial hurdles to be extended to
out-of-distribution (*OOD*) problem due to the distribution drift over
temporally evolving domains. Therefore, it is essential to enhance the
theoretical analyses on the model capacity and theoretical relation
among different temporal domain generalization models.
:::

::: {#S1.p5 .ltx_para .ltx_noindent}
To address all the above challenges, we propose a Temporal Domain
Generalization with [DR]{#S1.p5.1.1 .ltx_text .ltx_font_bold
.ltx_framed_underline}ift-[A]{#S1.p5.1.2 .ltx_text .ltx_font_bold
.ltx_framed_underline}ware dynam[I]{#S1.p5.1.3 .ltx_text .ltx_font_bold
.ltx_framed_underline}c neural [N]{#S1.p5.1.4 .ltx_text .ltx_font_bold
.ltx_framed_underline}etworks ([DRAIN]{#S1.p5.1.5 .ltx_text
.ltx_font_bold}) framework that solves all challenges above
simultaneously. Specifically, we propose a generic framework to
formulate temporal domain generalization by a Bayesian treatment that
jointly models the relation between data and model dynamics. To
instantiate the Bayesian framework, a recurrent graph generation
scenario is established to encode and decode the dynamic
graph-structured neural networks learned across different timestamps.
Such a scenario can achieve a fully time-sensitive model and can be
trained in an end-to-end manner. It captures the temporal drift of model
parameters and data distributions, and can predict the models in the
future *without* the presence of future data.
:::

::: {#S1.p6 .ltx_para .ltx_noindent}
[Our contributions include:]{#S1.p6.1.1 .ltx_text .ltx_font_bold}
[1)]{#S1.p6.1.2 .ltx_text .ltx_font_bold} We develop a novel and
adaptive temporal domain generalization framework that can be trained in
an end-to-end manner. [2)]{#S1.p6.1.3 .ltx_text .ltx_font_bold} We
innovatively treat the model as a dynamic graph and leverage graph
generation techniques to achieve a fully time-sensitive model.
[3)]{#S1.p6.1.4 .ltx_text .ltx_font_bold} We propose to use the
sequential model to learn the temporal drift adaptively and leverage the
learned sequential pattern to predict the model status on the future
domain. [4)]{#S1.p6.1.5 .ltx_text .ltx_font_bold} We provide theoretical
analysis on both uncertainty quantification and generalization error of
the proposed method. [5)]{#S1.p6.1.6 .ltx_text .ltx_font_bold} We
demonstrate our model's efficacy and superiority with extensive
experiments.
:::
:::

::: {#S2 .section .ltx_section .ltx_indent_first}
## [2 ]{.ltx_tag .ltx_tag_section}Related work {#related-work .ltx_title .ltx_title_section}

::: {#S2.p1 .ltx_para .ltx_noindent}
[Continuous Domain Adaptation.]{#S2.p1.1.1 .ltx_text .ltx_font_bold}
Domain Adaptation (DA) has received great attention from researchers in
the past
decade \[[3](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib3){.ltx_ref},
[10](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib10){.ltx_ref},
[35](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib35){.ltx_ref}\]
and readers may refer
to \[[38](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib38){.ltx_ref}\]
for a comprehensive survey. Under the big umbrella of DA, continuous
domain adaptation considers the problem of adapting to target domains
where the domain index is a continuous variable (temporal DA is a
special case when the domain index is 1D). Approaches to tackling such
problems can be broadly classified into three categories: (1) biasing
the training loss towards future data via transportation of past
data\[[13](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib13){.ltx_ref},
[30](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib30){.ltx_ref}\],
(2) using time-sensitive network parameters and explicitly controlling
their evolution along
time \[[16](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib16){.ltx_ref},
[17](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib17){.ltx_ref},
[25](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib25){.ltx_ref}\],
(3) learning representations that are time-invariant using adversarial
methods \[[36](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib36){.ltx_ref}\].
The first category augments the training data, the second category
reparameterizes the model, and the third category redesigns the training
objective. However, data may not be available for the target domain, or
it may not be possible to adapt the base model, thus requiring Domain
Generalization.
:::

::: {#S2.p2 .ltx_para .ltx_noindent}
[Domain Generalization (DG).]{#S2.p2.1.1 .ltx_text .ltx_font_bold} A
diversity of DG methods have been proposed in recent
years \[[27](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib27){.ltx_ref},
[26](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib26){.ltx_ref},
[20](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib20){.ltx_ref},
[2](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib2){.ltx_ref},
[1](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib1){.ltx_ref},
[7](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib7){.ltx_ref},
[28](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib28){.ltx_ref}\].
According
to \[[37](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib37){.ltx_ref}\],
existing DG methods can be categorized into the following three groups,
namely: (1) *Data manipulation:* This category of methods focuses on
manipulating the inputs to assist in learning general representations.
There are two kinds of popular techniques along this line: a). Data
augmentation \[[33](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib33){.ltx_ref},
[34](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib34){.ltx_ref}\],
which is mainly based on augmentation, randomization, and transformation
of input data; b). Data
generation \[[23](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib23){.ltx_ref},
[31](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib31){.ltx_ref}\],
which generates diverse samples to help generalization. (2)
*Representation learning:* This category of methods is the most popular
in domain generalization. There are two representative techniques: a).
Domain-invariant representation
learning \[[10](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib10){.ltx_ref},
[11](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib11){.ltx_ref}\],
which performs kernel, adversarial training, explicitly features
alignment between domains, or invariant risk minimization to learn
domain-invariant representations; b). Feature
disentanglement \[[21](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib21){.ltx_ref}\],
which tries to disentangle the features into domain-shared or
domain-specific parts for better generalization. (3) *Learning
strategy:* This category of methods focuses on exploiting the general
learning strategy to promote the generalization capability, e.g,
ensemble
learning \[[24](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib24){.ltx_ref}\],
meta-learning \[[7](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib7){.ltx_ref}\],
gradient
operation \[[14](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib14){.ltx_ref}\],
etc.
:::

::: {#S2.p3 .ltx_para .ltx_noindent}
Existing works above consider generalization across categorical domains,
while in this paper, we assume the domain index set is across time
(namely, temporal), and the domain shifts smoothly over time.
Unfortunately, there is only very little work under this setting. The
first work called *Sequential Learning Domain Generalization*
(S-MLDG) \[[19](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib19){.ltx_ref}\]
proposed a DG framework over sequential domains based on the idea of
meta-learning. A more recent work called *Gradient Interpolation*
(GI) \[[28](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib28){.ltx_ref}\]
proposes a temporal DG algorithm to encourage a model to learn functions
that can extrapolate well to the near future by supervising the
first-order Taylor expansion of the learned function. However, neither
work can adaptively learn the temporal drift across the domains while
keeping the strong expressiveness of the learned model.
:::
:::

## [3 ]{.ltx_tag .ltx_tag_section}Methodology {#methodology .ltx_title .ltx_title_section}

::: {#S3.p1 .ltx_para .ltx_noindent}
In this section, we first provide the problem formulation of temporal
domain generalization and then introduce our proposed framework,
followed by our theoretical analyses.
:::

::: {#S3.SS1 .section .ltx_subsection .ltx_indent_first}
### [3.1 ]{.ltx_tag .ltx_tag_subsection}Problem formulation {#problem-formulation .ltx_title .ltx_title_subsection}

::: {#S3.SS1.p1 .ltx_para .ltx_noindent}
[Temporal Domain Generalization.]{#S3.SS1.p1.15.1 .ltx_text
.ltx_font_bold} We consider prediction tasks where the data distribution
evolves with time. During training, we are given $T$ observed source
domains $\mathcal{D}_{1},\mathcal{D}_{2},\cdots,\mathcal{D}_{T}$ sampled
from distributions on $T$ arbitrary time points
$t_{1} \leq t_{2} \leq \cdots \leq t_{T}$, with each
$\mathcal{D}_{s} = \left\{ {{(\mathbf{x}_{i}^{(s)},y_{i}^{(s)})} \in {\mathcal{X}_{s} \times \mathcal{Y}_{s}}} \right\}_{i = 1}^{N_{s}}$,
$s = {1,2,\cdots,T}$ where $\mathbf{x}_{i}^{(s)}$, $y_{i}^{(s)}$ and
$N_{s}$ denotes the input feature, label and sample size at timestamp
$t_{s}$, respectively, and $\mathcal{X}_{s}$, $\mathcal{Y}_{s}$ denotes
the input feature space and label space at timestamp $t_{s}$,
respectively. The trained model will only be tested on some target
domain *in the future*, i.e., $\mathcal{D}_{T + 1}$ where
$t_{T + 1} \geq t_{T}$. Our setting further assumes the existence of
concept drift across different domains, i.e., the domain distribution is
changing across time by following some patterns. For example, if we
consider how personal income changes each year, we could find that the
average income typically increases by some (varying) ratio every year
due to the inflation. Similar patterns can also be found on the housing
price and education cost, etc.
:::

::: {#S3.SS1.p2 .ltx_para .ltx_noindent}
Our *goal* is to build a model that proactively captures the concept
drift. Given labeled data from the source domains
$\mathcal{D}_{1},\mathcal{D}_{2},\cdots,\mathcal{D}_{T}$, we learn the
mapping function
$g_{{\mathbf{ω}}_{s}}:{\mathcal{X}_{s}\rightarrow\mathcal{Y}_{s}}$ on
each domain $\mathcal{D}_{s}$, $s = {1,2,\cdots,T}$ where
${\mathbf{ω}}_{s}$ denotes the function parameters at timestamp $t_{s}$,
respectively, and then predict the dynamics across the parameters
${\mathbf{ω}}_{1},{\mathbf{ω}}_{2},\cdots,{\mathbf{ω}}_{T}$. Finally, we
predict the parameters ${\mathbf{ω}}_{T + 1}$ for the mapping function
$g_{{\mathbf{ω}}_{T + 1}}:{\mathcal{X}_{T + 1}\rightarrow\mathcal{Y}_{T + 1}}$
on the unseen future domain. As shown in Figure [[1]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#S1.F1 "Figure 1 ‣ 1 Introduction ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref},
due to the temporal drift in data distribution, e.g. the input features
such as Twitter user age distribution and number of tweets increase each
year, the prediction model is expected to evolve accordingly, e.g. the
magnitude of model parameter weights will decrease annually. Despite the
necessity, handling the above problem is an open research area due to
several existing challenges: 1) Difficulty in characterizing data
distribution drift as well as how it influences the model. 2) Lack of
expressiveness in automatically capturing the dynamics of how neural
network evolves across time. 3) Theoretical guarantee on model's
performance (e.g., generalization error, uncertainty) on future domains
is hard to obtain due to the unknown and (potentially) complicated
concept drift.
:::
:::

::: {#S3.SS2.p1 .ltx_para .ltx_noindent}
In this section, we introduce how we address the challenges mentioned
above. For the first challenge, we build a systematic Bayesian
probability framework to represent the concept drift over the domains,
which instantly differentiates our work from all existing methods in DG.
For the second challenge, we propose modeling a neural network with
changing parameters as a dynamic graph and achieving a temporal DG
framework that can be trained end-to-end by graph generation techniques.
We further improve the proposed method's generalization ability by
introducing a skip connection module over different domains. Finally, to
handle the last challenge, we explore theoretical guarantees of model
performance under the challenging temporal DG setting and provide
theoretical analyses of our proposed method, such as uncertainty
quantification and generalization error.
:::

::: {#S3.SS2.SSS1 .section .ltx_subsubsection .ltx_indent_first}
#### [3.2.1 ]{.ltx_tag .ltx_tag_subsubsection}A probabilistic view of concept drift in temporal domain generalization {#a-probabilistic-view-of-concept-drift-in-temporal-domain-generalization .ltx_title .ltx_title_subsubsection}

::: {#S3.SS2.SSS1.p1 .ltx_para .ltx_noindent}
To perform domain generalization over temporally indexed domains, we
need to capture the concept drift within a given time interval. From a
probabilistic point of view, for each domain $\mathcal{D}_{s}$,
$s = {1,2,\cdots,T}$, we can learn a neural network
$g_{{\mathbf{ω}}_{s}}$ by maximizing the conditional probability
$\Pr{(\left. {\mathbf{ω}}_{s} \middle| \mathcal{D}_{s} \right.)}$, where
${\mathbf{ω}}_{s}$ denotes the status of model parameters at timestamp
$t_{s}$. Due to the evolving distribution of $\mathcal{D}_{s}$, the
conditional probability
$\Pr{(\left. {\mathbf{ω}}_{s} \middle| \mathcal{D}_{s} \right.)}$ will
change over time accordingly. Our ultimate goal is to predict
${\mathbf{ω}}_{T + 1}$ given all training data
$\mathcal{D}_{1},\mathcal{D}_{2},\cdots,\mathcal{D}_{T}$
($\mathcal{D}_{1:T}$ for short), i.e.,
$\Pr{(\left. {\mathbf{ω}}_{T + 1} \middle| \mathcal{D}_{1:T} \right.)}$.
By the Law of total probability, we have

  -- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
     $$\begin{array}{r}                                                                                                                                                                                                                                                                                                                                                                    [(1)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
     {{{\Pr\left( {\mathbf{ω}}_{T + 1} \middle| \mathcal{D}_{1:T} \right)} = {\oint_{\Omega}{{\underset{\text{inference}}{\underbrace{\Pr ⁡\left( {\mathbf{ω}}_{T+1} \middle| {\mathbf{ω}}_{1:T},\mathcal{D}_{1:T} \right)}} \cdot \underset{\text{training}}{\underbrace{\Pr ⁡\left( {\mathbf{ω}}_{1:T} \middle| \mathcal{D}_{1:T} \right)}}}\operatorname{}{d{\mathbf{ω}}_{1:T}}}}}.}      
     \end{array}$$                                                                                                                                                                                                                                                                                                                                                                         
  -- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------

The first term in the integral of Eq. [[1]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#S3.E1 "1 ‣ 3.2.1 A probabilistic view of concept drift in temporal domain generalization ‣ 3.2 Proposed Method ‣ 3 Methodology ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}
represents the inference phase, i.e., how we predict the status of the
target neural network in the future (namely, ${\mathbf{ω}}_{T + 1}$)
given all history statuses, while the second term denotes the training
phase, i.e., how we leverage all source domains' training data
$\mathcal{D}_{1:T}$ to obtain the status of the neural network on each
source domain, namely ${\mathbf{ω}}_{1:T}$. By the chain rule of
probability, we can further decompose the training phase as follows:

  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ -- ----------------------------------------------------
     $$\begin{array}{cl}                                                                                                                                                                                                                                                                                 [(2)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
     {\Pr\left( {\mathbf{ω}}_{1:T} \middle| \mathcal{D}_{1:T} \right)} & {= {\prod_{s = 1}^{T}{\Pr\left( {\mathbf{ω}}_{s} \middle| {\mathbf{ω}}_{1:{s - 1}},\mathcal{D}_{1:T} \right)}}} \\                                                                                                              
      & {{= {{{\Pr\left( {\mathbf{ω}}_{1} \middle| \mathcal{D}_{1} \right)} \cdot {\Pr\left( {\mathbf{ω}}_{2} \middle| {\mathbf{ω}}_{1},\mathcal{D}_{1:2} \right)}}\operatorname{}\cdots\operatorname{}{\Pr\left( {\mathbf{ω}}_{T} \middle| {\mathbf{ω}}_{1:{T - 1}},\mathcal{D}_{1:T} \right)}}}.}      
     \end{array}$$                                                                                                                                                                                                                                                                                       
  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ -- ----------------------------------------------------

Here we assume for each time point $t_{s}$, the model parameter
${\mathbf{ω}}_{s}$ only depends on the current and previous domains
(namely, $\{\mathcal{D}_{i}:{i \leq s}\}$), and there is no access to
future data (even unlabeled). Now we can break down the whole training
process into $T - 1$ steps, where each step corresponds to learning the
model parameter on the new domain conditional on parameter statuses from
the history domains and training data, i.e.,
${{\Pr\left( {\mathbf{ω}}_{s + 1} \middle| {\mathbf{ω}}_{1:s},\mathcal{D}_{1:s},\mathcal{D}_{s + 1} \right)},{\forall s}} < T$.
:::
:::

::: {#S3.SS2.SSS2 .section .ltx_subsubsection .ltx_indent_first}
#### [3.2.2 ]{.ltx_tag .ltx_tag_subsubsection}Neural network with dynamic parameters {#neural-network-with-dynamic-parameters .ltx_title .ltx_title_subsubsection}

::: {#S3.SS2.SSS2.p1 .ltx_para .ltx_noindent}
Since the data distributions change temporally, the parameter
${\mathbf{ω}}_{s}$ in $g_{{\mathbf{ω}}_{s}}$ needs to be updated
accordingly to address the temporal drift across the domains. In this
work, we consider leveraging *dynamic graphs* to model the temporally
evolving neural networks in order to retain maximal expressiveness.
:::

::: {#S3.SS2.SSS2.p2 .ltx_para .ltx_noindent}
Intuitively, a neural network $g_{\mathbf{ω}}$ can be represented as an
edge-weighted graph $G = {(V,E,\psi)}$, where each node $v \in V$
represents a neuron of $g_{\mathbf{ω}}$ while each edge $e \in E$
corresponds to a connection between two neurons in $g_{\mathbf{ω}}$.
Moreover, given a connection $e$ between neuron $u$ and $v$, i.e.,
$e = {(u,v)} \in E$, function $\psi:{E\rightarrow{\mathbb{R}}}$ denotes
the weight parameter between these two neurons, i.e.,
${\psi\operatorname{}{(u,v)}} = w_{u,v}$, ${\forall{(u,v)}} \in E$.
Essentially,
${\mathbf{ω}} = {\psi\operatorname{}{(E)}} = {\{ w_{u,v}:{{(u,v)} \in E}\}}$
is a set of parameter values indexed by all edges in E and $\mathbf{ω}$
represents the entire set of parameters for neural network $g$. Notice
that we give a general definition of $g_{\mathbf{ω}}$ so that both
shallow models (namely, linear model) and deep neural networks (e.g.,
MLP, CNN, RNN, GNN) can be treated as special cases here. We aim to
characterize the potential drift across domains by optimizing and
updating the graph structure (i.e., edge weight) of $g_{\mathbf{ω}}$.
\[[39](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib39){.ltx_ref}\]
have proven that optimizing the graph structure of the neural network
could have a smaller search space and a more smooth optimization
procedure than exhaustively searching over all possible connectivity
patterns.
:::

::: {#S3.SS2.SSS2.p3 .ltx_para .ltx_noindent}
We consider the case where the architecture or topology of neural
network $g_{\mathbf{ω}}$ is given, i.e., $V$ and $E$ are fixed, while
the parameter $\mathbf{ω}$ is changing constantly w.r.t time point
$t_{s}$. In this sense, we can write
${\mathbf{ω}}_{s} = {\psi\operatorname{}{(\left. E \middle| s \right.)}}$
where $\psi{( \cdot |s)}$ (*abbrev.* $\psi_{s}$) depends only on time
point $t_{s}$. Now the triplet $G = {(V,E,\psi_{s})}$ defines a *dynamic
graph* with evolving edge weights.
:::
:::

::: {#S3.SS2.SSS3 .section .ltx_subsubsection .ltx_indent_first}
#### [3.2.3 ]{.ltx_tag .ltx_tag_subsubsection}End-to-end learning of concept drift {#end-to-end-learning-of-concept-drift .ltx_title .ltx_title_subsubsection}

::: {#S3.SS2.SSS3.p1 .ltx_para .ltx_noindent}
Given history statuses $\{{\mathbf{ω}}_{1:s}\}$ of the neural network
learned from $\{\mathcal{D}_{1:s}\}$, we aim at generalizing and
extrapolating ${\mathbf{ω}}_{s + 1}$ so that it produces good
performance on the new domain $\mathcal{D}_{s + 1}$ in an end-to-end
manner. In fact, by viewing the neural networks $\{{\mathbf{ω}}_{1:s}\}$
as dynamically evolving graphs, a natural choice is to characterize the
latent graph distribution of $\{{\mathbf{ω}}_{1:s}\}$ by learning from
its evolving trend. Consequently, $\mathbf{ω}$'s can be directly sampled
from the distribution for the prediction in future domains.
:::

::: {#S3.SS2.SSS3.p2 .ltx_para .ltx_noindent}
We characterize the latent distribution of $\{{\mathbf{ω}}_{1:s}\}$ as a
sequential learning process based on a recurrent architecture, and each
unit $f_{\theta}$ in the recurrent model is parameterized by $\theta$ to
generate ${\mathbf{ω}}_{s}$ by accounting for previous
$\{{\mathbf{ω}}_{i}:{i < s}\}$. Specifically, at each recurrent block
(i.e., time step) $t_{s}$, $f_{\theta}$ produces two outputs
$(m_{s},h_{s})$, where $m_{s}$ is the current memory state and $h_{s}$
is a latent probabilistic distribution (i.e., hidden output of
$f_{\theta}$) denoting the information carried from previous time steps.
The latent probabilistic distribution $h_{t}$ allows us to generate the
dynamic graph ${\mathbf{ω}}_{s}$ by a decoding function
$F_{\xi}\operatorname{}{( \cdot )}$. Intuitively, different from
existing works that train and regularize a neural network on single
domain \[[28](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib28){.ltx_ref}\],
here we focus on directly searching for distribution of networks with
"good architectures". Lastly, the sampled ${\mathbf{ω}}_{s}$ is encoded
by a graph encoding function $G_{\eta}\operatorname{}{( \cdot )}$, which
then serves as the input of next recurrent block. Such a recurrent model
is trained on a single domain $\mathcal{D}_{s}$ to generate
${\mathbf{ω}}_{s}$ for prediction by minimizing the empirical loss,
i.e.,
$\text{min}_{\theta,\xi,\eta}\operatorname{}{\sum_{i = 1}^{N_{s}}{\ell\operatorname{}\left( {g_{{\mathbf{ω}}_{s}}\operatorname{}{(\mathbf{x}_{i}^{(s)})}},y_{i}^{(s)} \right)}}$,
where $\ell\operatorname{}{( \cdot , \cdot )}$ can be cross-entropy for
classification or MSE for regression. The optimal ${\mathbf{ω}}_{s}$ on
domain $\mathcal{D}_{s}$ will then be fed into the next domain
$\mathcal{D}_{s + 1}$ along with the memory state $m_{s}$ as input to
guide the generation of ${\mathbf{ω}}_{s + 1}$ until the entire training
phase is done. For the inference phase, we feed the optimal parameters
from the last training domain, namely ${\mathbf{ω}}_{T}$, into the
encoding function and leverage the recurrent block, together with the
memory state $m_{T}$ to predict the latent vector on the future domain
$\mathcal{D}_{T + 1}$, followed by the decoding function to decode the
latent vector and generate the optimal parameters
${\mathbf{ω}}_{T + 1}$.
:::

::: {#S3.SS2.SSS3.p3 .ltx_para .ltx_noindent}
In our implementation, the encoding and decoding functions are
instantiated as MLPs. The total number of parameters of the encoding and
decoding functions is
$\mathcal{O}\operatorname{}{({{N\operatorname{}d} + C})}$, which is
*linear* in $N$. Here $N$ is the number of parameters in predictive
models (namely $\mathbf{ω}$), $d$ is the width (i.e., number of neurons)
of the last hidden layer of the encoding and decoding functions, and $C$
denotes the number of parameters for all the layers before the last for
the encoding and decoding functions. Additionally, in many situations,
the first few layers of representation learning could be shared. Hence,
we do not need to generate all the parameters in $\mathbf{ω}$, but just
the last few layers.
:::
:::

::: {#S3.SS2.SSS4 .section .ltx_subsubsection .ltx_indent_first}
#### [3.2.4 ]{.ltx_tag .ltx_tag_subsubsection}Less forgetting and better generalization {#less-forgetting-and-better-generalization .ltx_title .ltx_title_subsubsection}

::: {#S3.SS2.SSS4.p1 .ltx_para .ltx_noindent}
During the training of recurrent models, it is also likely to encounter
the performance degradation problem. Such a problem can be severe in
temporal DG since a more complicated concept correlation exists between
each domain. In addition, if the training procedure on each domain
$\mathcal{D}_{s}$ takes a large number of iterations to converge, we may
also observe the forgetting phenomenon (i.e., the recurrent model
$f_{\theta}$ will gradually focus on the current training domain and
have less generalization capability for future domains). To alleviate
such a phenomenon, we leverage a straightforward technique - skip
connection to bridge the training on $\mathcal{D}_{s}$ with previous
domains $\{\mathcal{D}_{1:{s - 1}}\}$. Specifically,

  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
     $${{\Phi\operatorname{}\left( {\mathbf{ω}}_{s},\left\{ {\mathbf{ω}}_{{s - \tau}:{s - 1}} \right\} \right)} ≔ {{\mathbf{ω}}_{s} + {\lambda \cdot {\sum_{i = {s - \tau}}^{s - 1}{\mathbf{ω}}_{i}}}}},$$      [(3)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------

where $\lambda$ is regularization coefficient and $\tau$ denotes the
size of the *sliding window*. The skip connection could enforce the
generated network parameters ${\mathbf{ω}}_{s}$ to contain part of
previous network's information, and the implementation of the
fixed-sized sliding window can better alleviate the potential drawback
of the computational cost. We summarize the overall generative process
in Appendix [[A.2]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.SS2 "A.2 Overall generation process ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}.
:::
:::
:::

### [3.3 ]{.ltx_tag .ltx_tag_subsection}Theoretical analysis {#theoretical-analysis .ltx_title .ltx_title_subsection}

::: {#S3.SS3.p1 .ltx_para .ltx_noindent}
In this section, we provide a theoretical analysis of our proposed
framework's performance in the target domain. Our analyses include
uncertainty quantification and generalization error. Uncertainty
characterizes the dispersion or error of an estimate due to the noise in
measurements and the finite size of data sets, and smaller uncertainty
means less margin of error over the model predictions. On the other
hand, generalization error measures how accurate the model's prediction
is on unseen data. Our analyses show that our proposed DRAIN achieves
[both better prediction accuracy as well as smaller margin of error on
target domain]{#S3.SS3.p1.1.1 .ltx_text .ltx_font_bold} compared with
online and offline DG baselines. All proofs can be found in the appendix
due to the limited space here.
:::

::: {#S3.SS3.p2 .ltx_para .ltx_noindent}
First, we introduce two DG methods, namely online baseline and offline
baseline as defined below:
:::

::: {#Thmdefinition1 .ltx_theorem .ltx_theorem_definition}
###### [[Definition 1]{#Thmdefinition1.1.1.1 .ltx_text .ltx_font_bold}]{.ltx_tag .ltx_tag_theorem}[.]{#Thmdefinition1.2.2 .ltx_text .ltx_font_bold} {#definition-1. .ltx_title .ltx_runin .ltx_title_theorem}

::: {#Thmdefinition1.p1 .ltx_para}
Given timestamp $t_{s + 1}$ and domains
$\mathcal{D}_{1},\mathcal{D}_{2},\cdots,\mathcal{D}_{s + 1}$, and model
parameter state from previous timestamp, namely ${\mathbf{ω}}_{s}$.
Define online model $\mathcal{M}_{\text{on}}$ and offline model
$\mathcal{M}_{\text{off}}$ as
$\omega_{s + 1} = {\text{argmax}_{\omega_{s + 1}}\operatorname{}{\Pr{(\left. \omega_{s + 1} \middle| {\omega_{s},\mathcal{D}_{s + 1}} \right.)}}}$
and
$\omega_{s + 1} = {\text{argmax}_{\omega_{s + 1}}\operatorname{}{\Pr{(\left. \omega_{s + 1} \middle| \mathcal{D}_{1:{s + 1}} \right.)}}}$,
respectively.
:::
:::

::: {#S3.SS3.p3 .ltx_para .ltx_noindent}
Offline method $\mathcal{M}_{\text{off}}$ is trained using ERM over all
source domains, while online method $\mathcal{M}_{\text{on}}$ considers
one-step finetuning over the model parameter on each new domain's
dataset. Both $\mathcal{M}_{\text{off}}$ and $\mathcal{M}_{\text{on}}$
are time-oblivious, i.e., unaware of the concept drift over time.
:::

     \end{aligned}$$                                                                                                                                                                                                                                                                                  
  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
:::

### [4.2 ]{.ltx_tag .ltx_tag_subsection}Quantitative analysis {#quantitative-analysis .ltx_title .ltx_title_subsection}

::: {#S4.SS2.p1 .ltx_para .ltx_noindent}
We firstly illustrate the performance of our proposed method against
comparison methods. The experiments are conducted in both classification
and regression tasks with the domain generalization setting, i.e.,

testing domain.
:::

::: {#S4.SS2.p2 .ltx_para .ltx_noindent}
As can be seen from Table [[1]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#S4.T1 "Table 1 ‣ 4 Experiment ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref},
DRAIN consistently achieves competitive results across most datasets.
Specifically, DRAIN excels the second-best approaches on Elec2 (CIDA),
House (GI) and Appliance (GI) by a great margin. The only exception is
the ONP dataset, where the Offline method achieves the best result and
all state-of-the-art methods cannot generalize well on unseen testing
domains since the ONP dataset does not exhibit a strong concept drift.
Additionally, all time-oblivious baselines perform rather
unsatisfactorily since they are not capable of handling the concept
drift of the data distribution. Both CDOT and CIDA can generate better
results than time-oblivious baselines, yet their generalization ability
on the unseen domains is still limited as the maintained time-invariant
representation in both methods cannot address the concept drift without
any data in the testing domain. As the only method that addresses the
temporal domain generalization problem, GI imposes a gradient
regularization with a non-parametric activation function to handle the
concept drift, which relies too much on the task-specific heuristic. In
contrast, DRAIN proposes to sequentially model each domain in an
end-to-end manner, which could address the concept drift more
inherently.
:::
:::

### [4.5 ]{.ltx_tag .ltx_tag_subsection}Ablation study {#ablation-study .ltx_title .ltx_title_subsection}

<figure id="S4.T2" class="ltx_table">
<figcaption><span class="ltx_tag ltx_tag_table">Table 2: </span>Ablation
study. Comparison of performance between our method and two alternatives
across two datasets for classification tasks and one datasets for
regression tasks.</figcaption>
</figure>

::: {#S4.SS5.p1 .ltx_para .ltx_noindent}
We further conduct ablation study on three datasets to evaluate the
effect of different components in DRAIN, and the results are exhibited
in Table [[2]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#S4.T2 "Table 2 ‣ 4.5 Ablation study ‣ 4 Experiment ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}.
Specifically, we remove the sequential learning model in DRAIN, and the
resulted ablated model ✗ RNN corresponds to the IncFinetune baseline
model. We also independently remove the skip connection module to let
the sequential learning model uniformally acquire information from all
previous domains, and the resulting model is named ✗ Skip.C.
:::

::: {#S4.SS5.p2 .ltx_para .ltx_noindent}
As shown in the table, yet each component can effectively contribute to
the overall model performance, modeling the temporal correlation between
all domains by a sequential model can provide a rather larger
performance gain. In addition, removing the skip connection in the
sequential learning model would make DRAIN hard to capture the long
range temporal dependency among domains since long range domain
information could potentially be forgotten during the model learning.
:::

## [5 ]{.ltx_tag .ltx_tag_section}Conclusion {#conclusion .ltx_title .ltx_title_section}

::: {#S5.p1 .ltx_para .ltx_noindent}
We tackle the problem of temporal domain generalization by proposing a
dynamic neural network framework. We build a Bayesian framework to model
the concept drift and treat a neural network as a dynamic graph to
capture the evolving pattern. We provide theoretical analyses of our
proposed method, such as uncertainty and generalization error, and
extensive empirical results to demonstrate the efficacy and efficiency
of our method compared with state-of-the-art DA and DG methods.
:::
:::

#### [A.1.1 ]{.ltx_tag .ltx_tag_subsubsection}Dataset Details {#a.1.1-dataset-details .ltx_title .ltx_title_subsubsection}

::: {#A1.SS1.SSS1.p1 .ltx_para .ltx_noindent}
We expand upon the datasets used for our experiments in this section. We
highlighted the sentence that describes the domain drift within each
dataset.
:::

::: {#A1.SS1.SSS1.p2 .ltx_para .ltx_noindent}
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I1.i1}

    ::: {#A1.I1.i1.p1 .ltx_para}
    [Rotated 2 Moons:]{#A1.I1.i1.p1.2.2 .ltx_text .ltx_font_bold} This
    is a variant of the 2-entangled moons dataset, with a lower moon and
    an upper moon labeled 0 and 1 respectively. Each moon consists of
    100 instances, and 10 domains are obtained by sampling 200 data
    points from the 2-Moons distribution, and rotating them
    counter-clockwise in units of $18^{\circ}$. Domains 0 to 8 (both
    inclusive) are our training domains, and domain 9 is for testing.
    *Each domain is obtained by rotating the moons counter-clockwise in
    units of $18^{\circ}$, hence the concept drift means the rotation of
    the moon-shape clusters.*
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I1.i2}

    ::: {#A1.I1.i2.p1 .ltx_para}
    [Rotated MNIST:]{#A1.I1.i2.p1.3.2 .ltx_text .ltx_font_bold} This is
    an adaptation of the popular MNIST digit
    dataset \[[5](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib5){.ltx_ref}\],
    where the task is to classify a digit from 0 to 9 given an image of
    the digit. We generate 5 domains by rotating the images in steps of
    15 degrees. To generate the $i$-th domain, we sample 1,000 images
    from the MNIST dataset and rotate them counter-clockwise by
    $15 \times$ i degrees. We take the first four domains as train
    domains and the fifth domain as test. *Similar to 2-Moons, each
    domain here is generated by rotating the images of digits by
    $15^{\circ}$, hence the concept drift means the rotation of the
    images.*
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I1.i3}

    ::: {#A1.I1.i3.p1 .ltx_para}
    [Online News Popularity:]{#A1.I1.i3.p1.1.1 .ltx_text .ltx_font_bold}
    This
    dataset \[[8](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib8){.ltx_ref}\]
    summarizes a heterogeneous set of features about articles published
    by Mashable in a period of two years. The goal is to predict the
    number of shares in social networks (popularity). We split the
    dataset by time into 6 domains and use the first 5 for training.
    *The concept drift is reflected in the change of time, but previous
    works have proven
    \[[28](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib28){.ltx_ref}\]
    the concept drift is not strong.*
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I1.i4}

    ::: {#A1.I1.i4.p1 .ltx_para}
    [Shuttle:]{#A1.I1.i4.p1.1.1 .ltx_text .ltx_font_bold} This dataset
    provides about 58,000 data points for space shuttles in flight. The
    task is multiclass classification with a heavy class imbalance. The
    dataset was divided into 8 domains based on the time points
    associated with points, with times between 30-70 being the train
    domains and 70 -80 being the test domain.
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I1.i5}

    ::: {#A1.I1.i5.p1 .ltx_para}
    [Electrical Demand]{#A1.I1.i5.p1.1.1 .ltx_text .ltx_font_bold} This
    contains information about the demand of electricity in a particular
    province. The task is, again binary classification, to predict if
    the demand of electricity in each period (of 30 mins) was higher or
    lower than the average demand over the last day. We consider two
    weeks to be one time domain, and train on 29 domains while testing
    on domain 30. *Each domain is generated by considering the demand of
    electricity within certain two weeks, so the domain drift can be
    regarded as how the electricity demand is changing seasonally.*
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I1.i6}

    ::: {#A1.I1.i6.p1 .ltx_para}
    [House Prices Dataset:]{#A1.I1.i6.p1.1.1 .ltx_text .ltx_font_bold}
    This dataset has housing price data from 2013-2019. This is a
    regression task to predict the price of a house given the features.
    We treat each year as a separate domain, but also give information
    about the exact date of purchase to the models. We take data from
    the year 2019 to be test data and prior data as training. *Similar
    to Elec2, the concept drift in this dataset is how the housing price
    changed from 2013-2019 for a certain region.*
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I1.i7}

    ::: {#A1.I1.i7.p1 .ltx_para .ltx_noindent}
    [Appliances Energy Prediction:]{#A1.I1.i7.p1.1.1 .ltx_text
    .ltx_font_bold} This
    dataset \[[4](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib4){.ltx_ref}\]
    is used to create regression models of appliances energy use in a
    low energy building. The data set is at 10 min for about 4.5 months
    in 2016, and we treat each half month as a single domain, resulting
    in 9 domains in total. The first 8 domains are used for training and
    the last one is for testing. *Similar to Elec2, the drift for this
    dataset corresponds to how the appliances energy usage changes in a
    low energy building over about 4.5 months in 2016.*
    :::
:::
:::

::: {#A1.SS1.SSS2 .section .ltx_subsubsection .ltx_indent_first}
#### [A.1.2 ]{.ltx_tag .ltx_tag_subsubsection}Details of Comparison Methods {#a.1.2-details-of-comparison-methods .ltx_title .ltx_title_subsubsection}

::: {#A1.SS1.SSS2.p1 .ltx_para .ltx_noindent}
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I2.i1}

    ::: {#A1.I2.i1.p1 .ltx_para}
    [Practical Baseline.]{#A1.I2.i1.p1.1.1 .ltx_text .ltx_font_bold}
    *1). Offline:* this is a time-oblivious model that is trained using
    ERM on all the source domains. *2). LastDomain:* this is a
    time-oblivious model that is trained using ERM on the last source
    domains. *3). IncFinetune:* we bias the training towards more recent
    data by applying the Baseline method described above on the first
    time point and then, fine-tuning with a reduced learning rate on the
    subsequent time points in sequential manner. This baseline
    corresponds to the online model we defined in
    Definition [[1]{.ltx_text
    .ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#Thmdefinition1 "Definition 1. ‣ 3.3 Theoretical analysis ‣ 3 Methodology ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}.
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I2.i2}

    ::: {#A1.I2.i2.p1 .ltx_para}
    [Continuous Domain Adaptation Methods.]{#A1.I2.i2.p1.1.1 .ltx_text
    .ltx_font_bold} *1). CDOT:* this model transports most recent
    labeled examples $\mathcal{D}^{T}$ to the future using a learned
    coupling from past data, and trains a classifier on them.. *2).
    CIDA:* this method is representative of typical domain erasure
    methods applied to continuous domain adaptation problems. *3).
    Adagraph:* This method makes the batch norm parameters
    time-sensitive and smooths them using a given kernel.
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I2.i3}

    ::: {#A1.I2.i3.p1 .ltx_para .ltx_noindent}
    [Temporal Domain Generalization Method.]{#A1.I2.i3.p1.1.1 .ltx_text
    .ltx_font_bold} *1). GI:* this method proposes a training algorithm
    to encourage a model to learn functions which can extrapolate well
    to the near future by supervising the first order Taylor expansion
    of the learnt function.
    :::
:::
:::

::: {#A1.SS1.SSS3 .section .ltx_subsubsection .ltx_indent_first}
#### [A.1.3 ]{.ltx_tag .ltx_tag_subsubsection}Parameter Setting {#a.1.3-parameter-setting .ltx_title .ltx_title_subsubsection}

::: {#A1.SS1.SSS3.p1 .ltx_para .ltx_noindent}
We use Adam optimizer for all our experiments, and the learning rate for
all datasets are uniformly set to be ${1\operatorname{}e} - 4$. All
experiments are conducted on a $64$-bit machine with $4$-core Intel Xeon
W-2123 @ 3.60GHz, 32GB memory and NVIDIA Quadro RTX 5000. We set
hyperparameters for each comparison method with respect to the
recommendation in their original paper, and we specify the architecture
as well as other details for each dataset's experiments as follows.

-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I3.i1}

    ::: {#A1.I3.i1.p1 .ltx_para}
    [2-Moons.]{#A1.I3.i1.p1.3.1 .ltx_text .ltx_font_bold} The number of
    layers in the LSTM is set to be $10$, and the network architecture
    of $g_{\omega_{t}}$ consists of 2 hidden layers, with a dimension of
    50 each. We use ReLU layer after each hidden layer and a Sigmoid
    layer after the output layer. The learning rate is set to be
    ${1\operatorname{}e} - 4$.
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I3.i2}

    ::: {#A1.I3.i2.p1 .ltx_para}
    [Rot-MNIST.]{#A1.I3.i2.p1.9.1 .ltx_text .ltx_font_bold} The number
    of layers in the LSTM is set to be $10$, and the network
    architecture of $g_{\omega_{t}}$ consists of $2$ convolution layers
    with kernel shape $3 \times 3$, and each convolution layer is
    followed by a max pooling layer with kernel size $2$ and stride
    $= 2$. The latent representation is then transformed by two linear
    layers with dimensions $256$ and $10$. We use ReLU layer after each
    hidden layer and a Sigmoid layer after the output layer. The
    learning rate is set to be ${1\operatorname{}e} - 3$.
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I3.i3}

    ::: {#A1.I3.i3.p1 .ltx_para}
    [ONP.]{#A1.I3.i3.p1.3.1 .ltx_text .ltx_font_bold} The number of
    layers in the LSTM is set to be $10$, and the network architecture
    of $g_{\omega_{t}}$ consists of 2 hidden layers with bias terms, and
    the dimensions of each layer are 20. We use ReLU layer after each
    hidden layer and a Sigmoid layer after the output layer. The
    learning rate is set to be ${1\operatorname{}e} - 4$.
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I3.i4}

    ::: {#A1.I3.i4.p1 .ltx_para}
    [Shuttle.]{#A1.I3.i4.p1.5.1 .ltx_text .ltx_font_bold} The number of
    layers in the LSTM is set to be $5$, and the network architecture of
    $g_{\omega_{t}}$ consists of $3$ hidden layers with bias terms, and
    the dimensions of each layer are $128$. We use ReLU layer after each
    hidden layer and a Sigmoid layer after the output layer. The

#### [A.1.6 ]{.ltx_tag .ltx_tag_subsubsection}Important Remarks {#a.1.6-important-remarks .ltx_title .ltx_title_subsubsection}

::: {#A1.SS1.SSS6.p1 .ltx_para .ltx_noindent}
In this section, we provide some important remarks over the proposed
DRAIN framework.

-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I4.i1}

    ::: {#A1.I4.i1.p1 .ltx_para}
    Graph generation can handle large graphs and there are a number of
    existing works that can handle large graphs. Our model is a general
    framework that can choose different graph generation methods as
    needed.
    :::
-   [[•]{.ltx_tag .ltx_tag_item}]{#A1.I4.i2}

    ::: {#A1.I4.i2.p1 .ltx_para .ltx_noindent}
    Neural networks are networks (i.e., graphs) of neurons, which have
    gained lots of research interest in recent years. Recent research
    (e.g., \[[39](https://ar5iv.labs.arxiv.org/html/2205.10664#bib.bib39){.ltx_ref}\])
    have found that the performance of neural network architectures is
    highly correlated with certain graph characteristics. In this work,
    we aim at characterizing the potential drift across the domains by
    optimizing and updating the graph structure of the neural network
    because optimizing the graph structure of a neural network has been
    proven to have a smaller search space and a more smooth optimization
    procedure than exhaustively searching over all possible connectivity
    patterns. Last but not least, our approach allows the entire neural
    network/model to change across time, which in turn maximizes our
    model's expressiveness.
    :::
:::
:::

.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#S3.F2 "Figure 2 ‣ 3.2 Proposed Method ‣ 3 Methodology ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}):

  -- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- --
     ${a_{1} = 0},{{m_{1} = {G_{0}\operatorname{}{(z)}}},{z \sim {\mathcal{N}\operatorname{}{(0,1)}}}}$                                                                                                                                                                
     ${a_{1} = {G_{\eta}\operatorname{}{(\omega_{1})}}},{{\omega_{1} \sim {F_{\xi}\operatorname{}{(h_{1})}}},{{(m_{1},h_{1})} = {f_{\theta}\operatorname{}{(m_{0},a_{0})}}}}$                                                                                          
     $\cdots$

     ${a_{2} = {G_{\eta}\operatorname{}{(\omega_{2})}}},{{\omega_{2} = {\Phi\operatorname{}{(\omega_{2},{\{\omega_{1}\}})}}},{{\omega_{2} = {F_{\xi}\operatorname{}{(h_{1})}}},{{(m_{2},h_{2})} = {f_{\theta}\operatorname{}{(m_{1},a_{1})}}}}}$                       
     $\cdots$                                                                                                                                                                                                                                                          
     ${a_{2} = {G_{\eta}\operatorname{}{(\omega_{2})}}},{{\omega_{2} = {\Phi\operatorname{}{(\omega_{2},{\{\omega_{1}\}})}}},{{\omega_{2} = {F_{\xi}\operatorname{}{(h_{1})}}},{{(m_{2},h_{2})} = {f_{\theta}\operatorname{}{(m_{2},a_{2})}}}}}$

     ${{a_{t} = {G_{\eta}\operatorname{}{(\omega_{2})}}},{{\omega_{t} = {\Phi\operatorname{}{(\omega_{t},{\{\omega_{{t - \tau}:{t - 1}}\}})}}},{{\omega_{2} = {F_{\xi}\operatorname{}{(h_{1})}}},{{(m_{t},h_{t})} = {f_{\theta}\operatorname{}{(m_{t},a_{t})}}}}}},$   
  -- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- --

\

where each $a_{i}$ denotes the input of $f_{\theta}$. In this work, we
utilize LSTM as the recurrent architecture, and $f_{\theta}$ becomes a
single LSTM unit. To initialize the whole generative process, we take a
random noise $z$ as input for the first domain $\mathcal{D}_{1}$, which
is drawn from a standard Gaussian distribution. The initial memory state
$m_{1}$ is also transformed from $z$ by an initial encoding function
$G_{0}\operatorname{}{( \cdot )}$.

::: {#A1.SS3 .section .ltx_subsection .ltx_indent_first}
### [A.3 ]{.ltx_tag .ltx_tag_subsection}Theory Proof {#a.3-theory-proof .ltx_title .ltx_title_subsection}

::: {#A1.SS3.p1 .ltx_para .ltx_noindent}
In this section, we provide the formal proof for Theorem 1 and Theorem 2
in our main context.
:::

::: {#A1.SS3.SSS1 .section .ltx_subsubsection .ltx_indent_first}
#### [A.3.1 ]{.ltx_tag .ltx_tag_subsubsection}Proof for Theorem 1 {#a.3.1-proof-for-theorem-1 .ltx_title .ltx_title_subsubsection}

::: {#A1.SS3.SSS1.27 .ltx_proof}
###### Proof. {#proof. .ltx_title .ltx_runin .ltx_font_italic .ltx_title_proof}

::: {#A1.SS3.SSS1.1.p1 .ltx_para .ltx_noindent}
By definition of the predictive distribution,
:::

::: {#A1.SS3.SSS1.2.p2 .ltx_para .ltx_noindent}
  -- -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
     $$\begin{aligned}                                                                                                                                                                                                                                                                                                                               [(5)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
     {{P\operatorname{}{(\left. \hat{y} \middle| {\mathbf{x}_{T + 1},\mathcal{D}_{1:T}} \right.)}} =} & {\oint_{\Omega}{P\operatorname{}{(\left. \hat{y} \middle| {\mathbf{x}_{T + 1},\omega_{T + 1}} \right.)}\operatorname{}P\operatorname{}{(\left. \omega_{T + 1} \middle| \mathcal{D}_{1:T} \right.)}\operatorname{}{d\omega_{T + 1}}}} \\      
      = & {\oint_{\Omega}{P\operatorname{}{(\left. \hat{y} \middle| {\mathbf{x}_{T + 1},\omega_{T + 1}} \right.)}\operatorname{}P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}\operatorname{}P\operatorname{}{(\left. \omega_{1:T} \middle| \mathcal{D}_{1:T} \right.)}\operatorname{}{d\omega_{1:{T + 1}}}}}               
     \end{aligned}$$                                                                                                                                                                                                                                                                                                                                 
  -- -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
:::

::: {#A1.SS3.SSS1.3.p3 .ltx_para .ltx_noindent}
Our goal is to prove that the variance of this predictive distribution
for our proposed method, online baseline and offline baseline follows
the inequality as in Theorem 1.
:::

::: {#A1.SS3.SSS1.4.p4 .ltx_para .ltx_noindent}
[Ours v.s. Online Baseline]{#A1.SS3.SSS1.4.p4.1.1 .ltx_text
.ltx_font_bold}
:::

::: {#A1.SS3.SSS1.5.p5 .ltx_para .ltx_noindent}
Here we prove that
${\text{Var}\operatorname{}{(\mathcal{M}_{\text{ours}})}} < {\text{Var}\operatorname{}{(\mathcal{M}_{\text{on}})}}$.
:::

::: {#A1.SS3.SSS1.6.p6 .ltx_para .ltx_noindent}
Notice that the first term on the right hand side of Eq. [[5]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E5 "5 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref},
namely
$P\operatorname{}{(\left. \hat{y} \middle| {\mathbf{x}_{T + 1},\omega_{T + 1}} \right.)}$,
corresponds to deployment of the model with parameter $\omega_{T + 1}$
on the future domain $\mathcal{D}_{T + 1}$, hence the variance of
$P\operatorname{}{(\left. \hat{y} \middle| {\mathbf{x}_{T + 1},\omega_{T + 1}} \right.)}$
only depends on the noise or randomness coming from $\mathbf{x}_{T + 1}$
as long as $\omega_{T + 1}$ is given. In other words, the uncertainty
coming from
$P\operatorname{}{(\left. \hat{y} \middle| {\mathbf{x}_{T + 1},\omega_{T + 1}} \right.)}$
can be cancelled for both methods since we are considering the same set
of domains. Now the problem reduces to prove that the variance of the
second and third terms on the right hand side of Eq. [[5]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E5 "5 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}
for our model is smaller than those for the online baseline.
:::

::: {#A1.SS3.SSS1.7.p7 .ltx_para .ltx_noindent}
By Bayesian statistics, we know that
:::

::: {#A1.SS3.SSS1.8.p8 .ltx_para .ltx_noindent}
  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
     $$\begin{array}{cl}                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          [(6)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
     {{P\operatorname{}{(\left. \omega_{1:T} \middle| \mathcal{D}_{1:T} \right.)}} =} & {\int_{\Theta}{{{{{{{{{P\operatorname{}{(\left. \omega_{1} \middle| \mathcal{D}_{1} \right.)}} \cdot P}\operatorname{}{(\left. \omega_{2} \middle| {\omega_{1},\mathcal{D}_{2},\theta_{0}} \right.)}} \cdot P}\operatorname{}{(\left. \theta_{1} \middle| {\omega_{1},\omega_{2},\theta_{0}} \right.)}} \cdot P}\operatorname{}{(\left. \omega_{3} \middle| {\omega_{2},\mathcal{D}_{3},\theta_{1}} \right.)}} \cdot P}\operatorname{}{(\left. \theta_{2} \middle| {\omega_{2},\omega_{3},\theta_{1}} \right.)}}} \\      
      & {{{\cdots\operatorname{}P\operatorname{}{(\left. \omega_{T} \middle| {\omega_{T - 1},\mathcal{D}_{T},\theta_{T - 2}} \right.)}} \cdot P}\operatorname{}{(\left. \theta_{T - 1} \middle| {\omega_{T - 1},\omega_{T},\theta_{T - 2}} \right.)}\operatorname{}d\operatorname{}\theta_{0:{T - 1}}}                                                                                                                                                                                                                                                                                                            
     \end{array}$$                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                
  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
:::

::: {#A1.SS3.SSS1.9.p9 .ltx_para .ltx_noindent}
where $\theta$ is the parameter of the parameterized function to
approximate the ground-truth drift of $\omega$, as defined in Assumption
 [[1]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#Thmassumption1 "Assumption 1 (Prior on Concept Drift). ‣ 3.3 Theoretical analysis ‣ 3 Methodology ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}.
For example,
$P\operatorname{}{(\left. \omega_{1} \middle| \mathcal{D}_{1} \right.)}$
denotes that we train the model on the very first domain and
$P\operatorname{}{(\left. \omega_{2} \middle| {\omega_{1},\mathcal{D}_{2},\theta_{0}} \right.)}$
denotes that we continue to train the model on the second domain but
with initialization of $\omega_{2}$ as
$q_{\theta_{0}}\operatorname{}{(\omega_{1})}$ where $\omega_{1}$ is
learned from the previous domain and $q_{\theta_{0}}$ is trying to
capture the conditional probability or drift between $\omega_{2}$ and
$\omega_{1}$, i.e.,
$P\operatorname{}{(\left. \omega_{2} \middle| \omega_{1} \right.)}$. In
our Bayesian framework, we treat $q_{\theta}$ as a learnable function
(e.g., LSTM unit in our proposed method) and we use subscript of
$\theta$ to differentiate the status of $\theta$ after the training on
each domain. In other words, $q_{\theta}$ will be updated after the
training on each domain (at least for our method). Notice that
$\theta_{0}$ always denotes the prior distribution as in
Assumption [[1]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#Thmassumption1 "Assumption 1 (Prior on Concept Drift). ‣ 3.3 Theoretical analysis ‣ 3 Methodology ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}.
:::

::: {#A1.SS3.SSS1.10.p10 .ltx_para .ltx_noindent}
By Bayes' rule, we have:
:::

::: {#A1.SS3.SSS1.11.p11 .ltx_para .ltx_noindent}
  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
     $${P\operatorname{}{(\left. \omega_{t + 1} \middle| {\omega_{t},\mathcal{D}_{t + 1},\theta_{t - 1}} \right.)}} \propto {\underset{\text{prior on~}\omega_{t + 1}}{\underbrace{P{({q_{\theta_{t-1}}{(\omega_{t})}})}}} \cdot \underset{\text{likelihood}}{\underbrace{P{(\left. \mathcal{D}_{t+1} \middle| \omega_{t+1} \right.)}}}}$$      [(7)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
:::

::: {#A1.SS3.SSS1.12.p12 .ltx_para .ltx_noindent}
where
$P\operatorname{}{({q_{\theta_{t - 1}}\operatorname{}{(\omega_{t})}})}$
can be regarded as the prior of $\omega_{t + 1}$ because as we mentioned
$q_{\theta_{t - 1}}$ denotes the initialization of $\omega_{t + 1}$
before we train the model on domain $\mathcal{D}_{t + 1}$, and
$P\operatorname{}{(\left. \mathcal{D}_{t + 1} \middle| \omega_{t + 1} \right.)}$
corresponds to the likelihood of training $\omega_{t + 1}$ on
$\mathcal{D}_{t + 1}$.
:::

::: {#A1.SS3.SSS1.13.p13 .ltx_para .ltx_noindent}
In addition,
:::

::: {#A1.SS3.SSS1.14.p14 .ltx_para .ltx_noindent}
  -- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
     $$\begin{array}{cl}                                                                                                                                                                                                                                     [(8)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
     {{P\operatorname{}{(\left. \theta_{t} \middle| {\omega_{t},\omega_{t + 1},\theta_{t - 1}} \right.)}} \propto} & {{{P\operatorname{}{(\theta_{t - 1})}} \cdot P}\operatorname{}{(\omega_{t},\left. \omega_{t + 1} \middle| \theta_{t} \right.)}} \\      
      \propto & {{{{{P\operatorname{}{(\theta_{t - 2})}} \cdot P}\operatorname{}{(\omega_{t - 1},\left. \omega_{t} \middle| \theta_{t - 1} \right.)}} \cdot P}\operatorname{}{(\omega_{t},\left. \omega_{t + 1} \middle| \theta_{t} \right.)}} \\            
      & \cdots \\                                                                                                                                                                                                                                            
      \propto & {\underset{\text{prior on~}\theta}{\underbrace{P{(\theta_{0})}}} \cdot \underset{\text{likelihood}}{\underbrace{\underset{i=1}{\overset{t}{\prod}}{P{(\omega_{i},\left. \omega_{i+1} \middle| \theta_{i} \right.)}}}}}                       
     \end{array}$$                                                                                                                                                                                                                                           
  -- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- ----------------------------------------------------
:::

::: {#A1.SS3.SSS1.15.p15 .ltx_para .ltx_noindent}
for any $t = {1,2,3,\cdots,{T - 1}}$. In the equation above, this time
the prior is over parameter $\theta$ and $\omega_{i}$, $\omega_{i + 1}$
can be regarded as the \"training data\" for $\theta_{i}$.
:::

::: {#A1.SS3.SSS1.16.p16 .ltx_para .ltx_noindent}
For the online baseline, since it only keeps one-step finetuning of the
model and does not learn how $\omega_{t}$ evolves, the $\theta_{t}$ for
the online baseline is always equal to the prior, i.e.
$\theta_{t} = \theta_{0}$. In other words,
${P\operatorname{}{({q_{\theta_{t - 1}}\operatorname{}{(\omega_{t})}})}} = {P\operatorname{}{({q_{\theta_{0}}\operatorname{}{(\omega_{t})}})}}$
and
${P\operatorname{}{(\left. \theta_{t} \middle| {\omega_{t},\omega_{t + 1},\theta_{t - 1}} \right.)}} = {P\operatorname{}{(\theta_{0})}}$,
$\forall t$ for the online baseline.
:::

::: {#A1.SS3.SSS1.17.p17 .ltx_para .ltx_noindent}
Since we follow the standard routine and assume all distributions are
Gaussian, by Bayesian Theorem, we know that the posterior distribution
always has variance smaller than the prior distribution, hence we know
that
:::

::: {#A1.SS3.SSS1.18.p18 .ltx_para .ltx_noindent}
  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ -- ----------------------------------------------------
     $${V\operatorname{}a\operatorname{}r\operatorname{}{(\left. \theta_{t} \middle| {\omega_{t},\omega_{t + 1},\theta_{t - 1}} \right.)}} < {V\operatorname{}a\operatorname{}r\operatorname{}{(\theta_{0})}}$$      [(9)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ -- ----------------------------------------------------
:::

::: {#A1.SS3.SSS1.19.p19 .ltx_para .ltx_noindent}
which proves that our method has smaller variance in terms of
Eq. [[8]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E8 "8 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}.
On the other hand, since the second term on the right hand side of
Eq. [[7]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E7 "7 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}
is the same for both methods, and for the first term
$P\operatorname{}{({q_{\theta_{t - 1}}\operatorname{}{(\omega_{t})}})}$,
by our Assumption  [[1]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#Thmassumption1 "Assumption 1 (Prior on Concept Drift). ‣ 3.3 Theoretical analysis ‣ 3 Methodology ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}
we know that for baseline
${\Pr{({q_{\theta_{t - 1}}\operatorname{}{(\omega_{t})}})}} = {\Pr{({q_{\theta_{0}}\operatorname{}{(\omega_{t})}})}}$
so the variance is basically $\sigma_{\theta_{0}}$. For our method,
after each training step across a new domain our $\theta$ will get
updated and achieve smaller variance (because of posterior variance of
Gaussian) so we also prove that our method has smaller variance in terms
of Eq. [[7]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E7 "7 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref}.
Two parts combined prove that our method has smaller variance in the
third term of Eq. [[5]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E5 "5 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref},
namely
$P\operatorname{}{(\left. \omega_{1:T} \middle| \mathcal{D}_{1:T} \right.)}$.
:::

::: {#A1.SS3.SSS1.20.p20 .ltx_para .ltx_noindent}
The last step is to compare the variance from the second term in
Eq. [[5]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E5 "5 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref},
namely
$P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}$.
For online baseline, basically it uses the parameter from the last
training domain, i.e., $\omega_{T}$ as the final model on the future
domain, i.e.,

  -- ----------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
     $${P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}} = {P\operatorname{}{({q_{\theta_{0}}\operatorname{}{(\omega_{T})}})}}$$      [(10)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- ----------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
:::

::: {#A1.SS3.SSS1.21.p21 .ltx_para .ltx_noindent}
On the other hand, for our method,

  -- --------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
     $${P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}} = {P\operatorname{}{({q_{\theta_{T - 1}}\operatorname{}{(\omega_{T})}})}}$$      [(11)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- --------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------

which has smaller variance due to the posterior variance of Gaussian.
:::

::: {#A1.SS3.SSS1.22.p22 .ltx_para .ltx_noindent}
All together we finish the proof for
${\text{Var}\operatorname{}{(\mathcal{M}_{\text{ours}})}} < {\text{Var}\operatorname{}{(\mathcal{M}_{\text{on}})}}$.
:::

::: {#A1.SS3.SSS1.23.p23 .ltx_para .ltx_noindent}
[Online Baseline v.s. Offline Baseline]{#A1.SS3.SSS1.23.p23.1.1
.ltx_text .ltx_font_bold}
:::

::: {#A1.SS3.SSS1.24.p24 .ltx_para .ltx_noindent}
This case is simpler to prove. Again, the first term on the right hand
side of Eq [[5]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E5 "5 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref},
namely
$P\operatorname{}{(\left. \hat{y} \middle| {\mathbf{x}_{T + 1},\omega_{T + 1}} \right.)}$
can be cancelled in this case. Moreover, the second term, namely
$P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}$
has the same variance for both baselines, i.e.,

  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
     $${V\operatorname{}a\operatorname{}r\operatorname{}{({P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}})}} = {V\operatorname{}a\operatorname{}r\operatorname{}{({P\operatorname{}{({q_{\theta_{0}}\operatorname{}{(\omega_{T})}})}})}} = \sigma_{\theta_{0}}$$      [(12)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------

This makes sense since two baselines do not learn the drift and the
uncertainty in predicting $\omega_{T + 1}$ based on $\omega_{T}$ is
always the same as the prior distribution of $\theta_{0}$.
:::

::: {#A1.SS3.SSS1.25.p25 .ltx_para .ltx_noindent}
Hence, it suffices to compare the uncertainty of the last term of
Eq. [[5]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#A1.E5 "5 ‣ Proof. ‣ A.3.1 Proof for Theorem 1 ‣ A.3 Theory Proof ‣ Appendix A Appendix ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref},
namely
$P\operatorname{}{(\left. \omega_{1:T} \middle| \mathcal{D}_{1:T} \right.)}$.
Recall that

  -- ------------------------------------------------------------------------------------------------------------------------------------------------ -- -----------------------------------------------------
     $$\begin{array}{cl}                                                                                                                                 [(13)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
      & {\mathcal{M}_{\text{on}}:\mspace{23mu}\omega_{t + 1} = \text{argmax}_{\omega_{t + 1}}P{(\omega_{t + 1}|\omega_{t},\mathcal{D}_{t + 1})}} \\      
      & {\mathcal{M}_{\text{off}}:\mspace{23mu}\omega_{t + 1} = \text{argmax}_{\omega_{t + 1}}P{(\omega_{t + 1}|\mathcal{D}_{1:{t + 1}})}}               
     \end{array}$$                                                                                                                                       
  -- ------------------------------------------------------------------------------------------------------------------------------------------------ -- -----------------------------------------------------
:::

::: {#A1.SS3.SSS1.26.p26 .ltx_para .ltx_noindent}
For offline baseline, we are using all dataset so far, namely
$\mathcal{D}_{1:{t + 1}}$ to train the model while the online baseline
only uses $\mathcal{D}_{t + 1}$. Since in this paper, we are considering
domain generalization with temporal concept drift, i.e., for each
$i \neq j$ we have $\mathcal{D}_{i} \neq \mathcal{D}_{j}$ (otherwise we
marge them), the randomness of $\bigcup_{i = 1}^{t + 1}\mathcal{D}_{i}$
is at least as large as that of $\mathcal{D}_{t + 1}$ alone, i.e.,
${V\operatorname{}a\operatorname{}r\operatorname{}{({\bigcup_{i = 1}^{t + 1}\mathcal{D}_{i}})}} \geq {V\operatorname{}a\operatorname{}r\operatorname{}{(\mathcal{D}_{t + 1})}}$.
Again, by Bayes' rule, we can finish the proof.
:::

::: {#A1.SS3.SSS1.27.p27 .ltx_para}
∎
:::
:::
:::

::: {#A1.SS3.SSS2 .section .ltx_subsubsection .ltx_indent_first}
#### [A.3.2 ]{.ltx_tag .ltx_tag_subsubsection}Proof of Theorem 2 {#a.3.2-proof-of-theorem-2 .ltx_title .ltx_title_subsubsection}

::: {#A1.SS3.SSS2.13 .ltx_proof}
###### Proof. {#proof.-1 .ltx_title .ltx_runin .ltx_font_italic .ltx_title_proof}

::: {#A1.SS3.SSS2.1.p1 .ltx_para .ltx_noindent}
Recall that
:::

::: {#A1.SS3.SSS2.2.p2 .ltx_para .ltx_noindent}
  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ -- -----------------------------------------------------
     $${e\operatorname{}r\operatorname{}r} ≔ {\ell\operatorname{}{({{\mathbb{E}}\operatorname{}{\lbrack{P\operatorname{}{(\left. \hat{y} \middle| {\mathbf{x}_{T + 1},\mathcal{D}_{1:T}} \right.)}}\rbrack}},y_{T + 1})}}$$      [(14)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ -- -----------------------------------------------------
:::

::: {#A1.SS3.SSS2.3.p3 .ltx_para .ltx_noindent}
First denote the ground-truth $\theta$ for the parameterized function
$q_{\theta}$ as $q_{\theta^{*}}$. The general idea of the proof below is
to show that our method can achieve $\theta$ that is closer to the
ground-truth $\theta^{*}$, i.e., learning the dynamics of
${\{\omega_{t}\}}_{t = 1}^{T}$ more accurately.
:::

::: {#A1.SS3.SSS2.4.p4 .ltx_para .ltx_noindent}
Notice that

  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
     $${{\mathbb{E}}\operatorname{}{\lbrack{P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}}\rbrack}} = {{\mathbb{E}}\operatorname{}{\lbrack{q_{\theta}\operatorname{}{(\omega_{T})}}\rbrack}}$$      [(15)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
:::

::: {#A1.SS3.SSS2.5.p5 .ltx_para .ltx_noindent}
By Assumption [[1]{.ltx_text
.ltx_ref_tag}](https://ar5iv.labs.arxiv.org/html/2205.10664#Thmassumption1 "Assumption 1 (Prior on Concept Drift). ‣ 3.3 Theoretical analysis ‣ 3 Methodology ‣ Temporal Domain Generalization with Drift-Aware Dynamic Neural Network"){.ltx_ref},
we know that by learning the conditional probability
$P\operatorname{}{(\left. \omega_{t + 1} \middle| \omega_{t} \right.)}$
our method can update $\theta$ of $q_{\theta}$. Assume that after
training on the first $T$ domains, our model can achieve error as
defined by:

  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
     $${{\mathbb{E}}\operatorname{}{\lbrack{\|{{q_{\theta_{T - 1}}\operatorname{}{(\omega_{T})}} - {q_{\theta^{*}}\operatorname{}{(\omega_{T})}}}\|}_{2}\rbrack}} < {{\mathbb{E}}\operatorname{}{\lbrack{\|{{q_{\theta_{0}}\operatorname{}{(\omega_{T})}} - {q_{\theta^{*}}\operatorname{}{(\omega_{T})}}}\|}_{2}\rbrack}}$$      [(16)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
:::

::: {#A1.SS3.SSS2.6.p6 .ltx_para .ltx_noindent}
where the left hand side denotes the expected error between the
predicted $\omega_{T + 1}$ and the ground-truth for our method while the
right hand side denotes that for the online baseline.
:::

::: {#A1.SS3.SSS2.7.p7 .ltx_para .ltx_noindent}
With some mild assumptions over the neural network, i.e., Lipschitz
continuity of the neural network w.r.t the parameter, we can measure
that the prediction error for our model is smaller than that of online
baseline.
:::

::: {#A1.SS3.SSS2.8.p8 .ltx_para .ltx_noindent}
On the other hand, for the comparison between two baselines, we consider
a simpler case where the drift of $\omega_{t}$ is monotonic but our
proof can be generalized to other cases easily.
:::

::: {#A1.SS3.SSS2.9.p9 .ltx_para .ltx_noindent}
As can be shown,

  -- -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
     $$\begin{array}{cl}                                                                                                                                                                                                                                                                                                                                                                       [(17)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
      & {{\text{Online baseline:}\quad{{\mathbb{E}}\operatorname{}{\lbrack{P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}}\rbrack}}} = {{\mathbb{E}}\operatorname{}{\lbrack{q_{\theta_{0}}\operatorname{}{(\omega_{T})}}\rbrack}} = \omega_{T}} \\                                                                                                                    
      & {{\text{Offline baseline:}\quad{{\mathbb{E}}\operatorname{}{\lbrack{P\operatorname{}{(\left. \omega_{T + 1} \middle| \omega_{1:T} \right.)}}\rbrack}}} = {{\mathbb{E}}\operatorname{}{\lbrack{q_{\theta_{0}}\operatorname{}{(\overline{\omega})}}\rbrack}} = {{\mathbb{E}}\operatorname{}{\lbrack{P\operatorname{}{(\left. \omega \middle| \mathcal{D}_{1:T} \right.)}}\rbrack}}}      
     \end{array}$$                                                                                                                                                                                                                                                                                                                                                                             
  -- -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
:::

::: {#A1.SS3.SSS2.10.p10 .ltx_para .ltx_noindent}
If we denote a distance function over the domains, as $d$, we assume
that
:::

::: {#A1.SS3.SSS2.11.p11 .ltx_para .ltx_noindent}
  -- ------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
     $${d\operatorname{}{(\mathcal{D}_{t + 1},\mathcal{D}_{T + 1})}} < {d\operatorname{}{(\mathcal{D}_{t},\mathcal{D}_{T + 1})}}$$      [(18)]{.ltx_tag .ltx_tag_equation .ltx_align_right}
  -- ------------------------------------------------------------------------------------------------------------------------------- -- -----------------------------------------------------
:::

::: {#A1.SS3.SSS2.12.p12 .ltx_para .ltx_noindent}
By the monotonic assumption, the distribution of each
$\mathcal{D}_{1:T}$ is changing along a certain direction. Hence, among
them $\mathcal{D}_{T}$ has the distribution most close to that of
$\mathcal{D}_{T + 1}$. In other words, the online baseline finetunes the
model so its $\omega_{T}$ is leaning towards the last domain while the
offline baseline is using the averaged domains to train the model, which
finishes the proof.
:::

::: {#A1.SS3.SSS2.13.p13 .ltx_para}
∎
:::
